## "Full Nowcasting" for Market Data Outlier Detection

This is a great concept to understand — and highly relevant to your problem. Let me break it all down.

---

### What Is "Full Nowcasting"?

**Nowcasting** originally comes from macroeconomics: predicting the *present* value of a variable (e.g., GDP) before official data is released, using high-frequency correlated indicators.

In your market data DQ context, **"full nowcasting"** means:

> *"Given everything else observable right now — all correlated peer instruments, market regime, term structure shape — what should this risk factor be?"*

If the **observed value diverges significantly from the nowcast**, it's flagged as an outlier.

The word **"full"** means you use the **entire cross-sectional information set** simultaneously, not just the factor's own time series history. This is the key distinction from pure univariate z-score checks.

---

### The Core Idea — Why It's Powerful for Your Problem

A classic 4-sigma check asks: *"Is this value unusual relative to its own history?"*

Nowcasting asks: *"Is this value inconsistent with everything else in the market right now?"*

| Scenario | 4σ Check | Nowcasting |
|---|---|---|
| Large real move (stress event) | ❌ False alarm | ✅ Peers also moved, consistent |
| Small data error (fat finger) | ✅ May miss it | ✅ Inconsistent with peers |
| Curve inversion anomaly | ❌ Each tenor looks fine individually | ✅ Caught via curve shape |
| Stale price (no update) | ❌ Misses it entirely | ✅ Peers moved, this didn't |

This is exactly why it dominates pure time-series approaches for your use case.

---

### The Mechanism & Algorithm

The full pipeline has three conceptual steps:

**Step 1 — Build a Nowcast Model**

For each risk factor $X_i$, train a model:

$$\hat{X}_i = f(X_{\text{peers}}, \text{regime indicators}, \text{term structure features})$$

Where peers = same asset class, same curve, same tenor bucket, same currency, etc.

**Step 2 — Compute Residuals**

$$r_i = X_i^{\text{observed}} - \hat{X}_i^{\text{nowcast}}$$

**Step 3 — Score & Flag**

Normalize residuals by a rolling conditional volatility estimate:

$$z_i = \frac{r_i}{\hat{\sigma}_i(\text{regime})}$$

Flag where $|z_i| > \tau$ (threshold varies by asset class and regime).

---

### Implementation Options — From Simple to Full

**Level 1 — Linear Factor Nowcast (PCA)**
The simplest version. For a curve (e.g., IR rates), extract 3 principal components (level, slope, curvature). Reconstruct each tenor from the factors. Large reconstruction error = outlier.

```
Fit PCA on peer group → project observation → residual → flag
```

Very fast. Works well for IR rates, IR vols, FX rates.

**Level 2 — Cross-Sectional Regression Nowcast**
For each risk factor, train a regression (linear or XGBoost) that predicts it from its peer group at the same timestamp. This is the practical sweet spot.

```
For each RF_i:
    features = [peer RF values at same timestamp, CDX/VIX regime, day-of-week, ...]
    target = RF_i
    Train model → predict → residual → flag
```

**Level 3 — Autoencoder (Neural Network)**
An encoder-decoder network that compresses the full peer group into a latent representation and reconstructs each factor. Reconstruction error = anomaly score. Works well for high-dimensional correlated surfaces (vol surfaces, CDS curves).

**Level 4 — State-Space / Kalman Filter**
Combines time-series dynamics with cross-sectional consistency. Most sophisticated but hardest to calibrate. Overkill for triage purposes.

“Full nowcasting” is probably the wrong tool for the core problem you described.

In most technical literature, nowcasting means estimating the current value of something that is only partially observed or published with delay, usually by combining timely inputs in a state-space or dynamic-factor framework, often with Kalman filtering. That is common in macroeconomics, demand sensing, weather, and mixed-frequency forecasting. The New York Fed’s nowcast and ECB/IMF work are classic examples. It is about inferring the present, not primarily about deciding whether a market data point is bad. 

So if someone told you “use full nowcasting” for market-data DQ, the useful interpretation is probably this: build a model that predicts what the current observation should roughly look like, given its own history, related curves, peer instruments, and current market context, then flag large residuals. That is a valid component in an anomaly framework. But it is not a magic, standalone answer, and it is not the normal name for a complete outlier-detection solution. It is better thought of as one signal inside a broader triage stack. 

The blunt answer to your bigger question is this: no, there is no single model that will reliably catch 95% of real issues across all asset classes and market data types without a costly labeling and governance setup. Time-series anomaly detection is highly context-dependent, and the research literature keeps stressing that performance varies a lot with data regime, anomaly type, and evaluation setup. Broad surveys and benchmarking work make that point very clearly. 

What usually gives the best bang for the buck in your setting is not “the best model per asset class.” It is a small, robust, layered system:

First, use simple robust statistical screens as the universal baseline. Median/MAD-based scoring is attractive because it is much less distorted by heavy tails and prior outliers than mean/std thresholds. NIST explicitly describes MAD as a robust outlier statistic and shows the standard scaling with 0.6745. For fat-tailed financial data, this is a much better default than naive 4-sigma. 

Second, add one general unsupervised learner such as Isolation Forest for multivariate context. Isolation Forest is appealing because it is designed for anomaly detection, scales well, and works by recursively partitioning the data so anomalies tend to be isolated in fewer splits. Scikit-learn’s documentation describes that mechanism directly. It is usually a better bank-scale default than trying to hand-tune many exotic models class by class. 

Third, add a forecast or nowcast residual signal only where it buys you something. For example, predict today’s 5Y BBB USD OAS change from its own lagged behavior, adjacent tenors, rating buckets, sector peers, index levels, and broad risk indicators. Then score the residual. This catches “value inconsistent with context,” which pure point-outlier methods miss. In other words, nowcasting is useful when your anomaly is “this print is not plausible given the rest of the curve/surface/market.” That is especially relevant for curves, surfaces, and cross-sectional structures. 

Fourth, combine the above with a triage layer, not a hard one-model decision. The anomaly literature is broad because different methods catch different anomaly shapes. That is why your instinct about a multi-model, context-aware, smart triage layer is directionally right. 

What nowcasting actually is

At a conceptual level, nowcasting estimates the current hidden or incompletely observed state from available data. In the classic setup:

You assume there is a latent state or factor driving several observed series.

New observations arrive with different lags or frequencies.

A state-space model updates the latent estimate as new “news” arrives.

Kalman filtering/smoothing is used to combine prior state estimates with noisy new information.


That is why dynamic factor models and Kalman filters appear so often in nowcasting references. Statsmodels’ DynamicFactorMQ is a good public example: it supports mixed-frequency nowcasting and explicitly uses state-space machinery; the New York Fed describes its nowcast as Bayesian dynamic factor modeling plus Kalman filtering. 

A simplified financial adaptation would look like this:

Observed series:

instrument level returns or shifts

neighboring tenor/rating/sector points

broad market variables like rates, CDS indices, SPX, VIX, MOVE, FX, commodity anchors

liquidity/activity proxies if available


Latent state:

“true fair move” for that risk factor today


Mechanism:

1. fit a model on history


2. produce expected current value or expected move


3. compare actual vs expected


4. score the residual


5. combine residual score with robust statistical score and peer-consistency score



That is the useful idea behind applying nowcasting to DQ.

Is there a best model per asset class?

Usually not in a stable, universal sense. The better framing is:

Univariate heavy-tailed series: robust rolling MAD, robust z-score, quantile bands.

Cross-sectional and multivariate context: Isolation Forest or similar.

Curves and surfaces: residual models plus shape checks such as monotonicity, smoothness, curvature jumps, neighbor consistency.

Highly structured markets: peer-group deviation checks often outperform fancy black-boxes because they encode domain structure directly.


That is consistent with the survey literature: anomaly-detection success depends on the anomaly definition, the data structure, and what “normal” means in that domain. 

Can any model catch 95% of real issues?

Not credibly as a general promise.

You might reach very high recall in a narrow class, but then false positives often explode. In your environment, “real issue” itself is ambiguous because many large market moves are genuine, and many bad prints can look superficially plausible. Without high-quality labels from SMEs or downstream evidence, any 95% claim should be treated with skepticism. Benchmarking papers on time-series anomaly detection have also warned that evaluation practices can be misleading and non-transferable. 

A more realistic target is:

high recall for severe, obvious breaks

moderate recall for subtle issues

strong prioritization so humans only review the top slice

progressive learning from adjudications and downstream corrections
